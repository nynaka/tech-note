Hyper-V (Windows 11)
===

## 1. 前提条件の確認

- Windows 11 Proエディション（HomeエディションではHyper-Vを利用できません）
- CPUが仮想化をサポートしていること（Intel VT-x/AMD-V）
- BIOSで仮想化が有効になっていること
- RAM 4GB以上（推奨8GB以上）
- ストレージ容量に十分な空き

## 2. Hyper-Vの有効化

### 方法1: コントロールパネルから有効化

1. 「スタート」メニューを開き、「設定」をクリックします
2. 「アプリ」→「オプション機能」→「その他のWindows機能」をクリックします
3. 表示されたリストから「Hyper-V」のチェックボックスにチェックを入れます
4. 「OK」をクリックします
5. システムの再起動を求められたら再起動します

### 方法2: PowerShellを使用して有効化

1. 「スタート」メニューを右クリックし、「Windows PowerShell (管理者)」または「ターミナル (管理者)」を選択します
2. 以下のコマンドを入力して実行します：

```powershell
Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Hyper-V -All
```

3. システムの再起動を求められたら「Y」を入力して再起動します

## 3. Hyper-Vマネージャーの起動

1. 「スタート」メニューを開き、「Hyper-V マネージャー」を検索して起動します
2. または、「Windowsキー + R」を押して「virtmgmt.msc」と入力し、「OK」をクリックします

## 4. 仮想スイッチの作成

1. Hyper-Vマネージャーの右側のアクションパネルから「仮想スイッチマネージャー」をクリックします
2. 「新しい仮想ネットワークスイッチ」を選択します
3. 「外部」を選択して「作成」をクリックします
4. 名前を入力（例：「External Network」）し、使用するネットワークアダプターを選択します
5. 「OK」をクリックして仮想スイッチを作成します

## 5. Ubuntu Linux仮想マシンのインストール手順

### 5.1 Ubuntu ISOファイルのダウンロード

1. 公式Ubuntuウェブサイト（[https://ubuntu.com/download/desktop](https://ubuntu.com/download/desktop)）からUbuntu DesktopのISOファイルをダウンロードします
2. 最新のLTS（Long Term Support）バージョンを選択することをお勧めします

### 5.2 仮想マシンの作成

1. Hyper-Vマネージャーで右側のアクションパネルから「新規」→「仮想マシン」をクリックします
2. 「新しい仮想マシンウィザード」が開きます
3. 「次へ」をクリックします
4. 仮想マシンの名前を入力（例：「Ubuntu」）し、必要に応じて保存場所を変更して「次へ」をクリックします
5. 世代を選択します（Ubuntu 18.04以降は「第2世代」を推奨、それ以前は「第1世代」）
6. メモリを割り当てます（Ubuntuには最低2GB、推奨4GB以上）
7. 先ほど作成した仮想スイッチを選択します
8. 仮想ハードディスクの設定で、サイズを指定します（推奨30GB以上）
9. 「インストールオプション」で「オペレーティングシステムを後でインストールする」を選択します
10. 「完了」をクリックして仮想マシンを作成します

### 5.3 仮想マシンの設定調整

1. 作成した仮想マシンを右クリックして「設定」を選択します
2. 第2世代を選択した場合は、「セキュリティ」→「セキュアブート」のチェックを外します
3. 「プロセッサ」で必要に応じてCPUコア数を増やします（推奨2コア以上）
4. 「SCSI コントローラー」→「DVDドライブ」→「メディア」→「イメージファイル」を選択し、ダウンロードしたUbuntu ISOファイルを指定します
5. 「OK」をクリックして設定を保存します

### 5.4 Ubuntuのインストール

1. 仮想マシンを選択し、「接続」をクリックして起動します
2. 「起動」ボタンをクリックします
3. Ubuntuのインストーラーが起動したら、言語を選択します
4. 「Ubuntu をインストール」をクリックします
5. キーボードレイアウトを選択します
6. 「通常のインストール」を選択します
7. 「ディスクを消去してUbuntuをインストール」を選択します
8. 「インストール」をクリックして確認ダイアログで「続ける」をクリックします
9. タイムゾーンを選択します
10. ユーザー名、コンピュータ名、パスワードを設定します
11. インストールが完了するまで待ちます
12. 「今すぐ再起動する」をクリックします

### 5.5 仮想マシンの最適化

1. Ubuntuが起動したら、ターミナルを開きます
2. システムを最新の状態に更新します：

```bash
sudo apt update
sudo apt upgrade -y
```

3. Hyper-V統合サービスをインストールします（最新のUbuntuでは通常自動インストールされています）：

```bash
sudo apt install hyperv-daemons -y
```

4. 再起動して変更を適用します：

```bash
sudo reboot
```

## 6. トラブルシューティング

### 一般的な問題と解決策

- **Hyper-Vを有効にできない場合：** BIOSで仮想化が有効になっているか確認してください
- **仮想マシンが起動しない：** セキュアブートの設定を確認してください
- **ネットワーク接続の問題：** 仮想スイッチの設定を確認し、必要に応じて再作成してください
- **パフォーマンスの問題：** 仮想マシンに割り当てるリソース（メモリ、CPU）を調整してください

### Hyper-V関連のコマンド

```powershell
# Hyper-Vのステータス確認
Get-WindowsOptionalFeature -Online -FeatureName Microsoft-Hyper-V

# すべての仮想マシンのリスト表示
Get-VM

# 特定の仮想マシンの起動
Start-VM -Name "Ubuntu"

# 特定の仮想マシンの停止
Stop-VM -Name "Ubuntu"

# 特定の仮想マシンの状態確認
Get-VMMemory -VMName "Ubuntu"
Get-VMProcessor -VMName "Ubuntu"
```

## 7. まとめ

これでWindows 11 Pro環境にHyper-Vを有効化し、Ubuntu Linuxの仮想マシンをインストールする手順は完了です。この環境を使って様々な検証やテストを行うことができます。仮想マシンのスナップショット機能を活用すると、システムの状態を保存して後で復元することも可能です。
