# Gemini CLI コンテキスト

このリポジトリは、[Docusaurus 3](https://docusaurus.io/) を使用して構築された技術ドキュメントサイトを含んでいます。Linux、Docker、サーバー管理、AI ツールなど、様々な技術に関する個人的なナレッジベースとして機能します。

## プロジェクト構成

- `docs/`: メインのドキュメントコンテンツ。ルートパス（`/`）で提供されます。
  - カテゴリ別に整理されています（例: `linux/`、`docker/`、`server/`、`tool/`、`windows/`）。
  - 各カテゴリには通常、サイドバーのメタデータを定義する `_category_.json` ファイルが含まれます。
- `draft/`: 作業中のドキュメント。`/draft` パスで提供されます。
- `src/`: カスタム React コンポーネントと CSS。
- `static/`: 画像やファビコンなどの静的アセット。
- `docusaurus.config.ts`: サイドバーのソートロジックやプラグイン定義を含む、サイトのメイン設定ファイル。
- `sidebars.ts`: 自動生成サイドバーの設定。

## 主要技術

- **Docusaurus 3.x**: 静的サイトジェネレーター。
- **React 19**: UI ライブラリ。
- **TypeScript**: 型付き JavaScript。
- **Mermaid**: Markdown 内のダイアグラムサポート。
- **MDX**: React コンポーネントをサポートした Markdown。

## ビルドと実行

プロジェクトは依存関係の管理に `npm` を使用します。

- **開発サーバーの起動**: `npm start`
- **本番用ビルド**: `npm run build`
- **型チェック**: `npm run typecheck`
- **キャッシュのクリア**: `npm run clear`

### デプロイ
サイトは GitLab Pages を通じてデプロイされるように設定されています。
- **CI/CD**: `.gitlab-ci.yml` がデプロイパイプラインを定義しており、`main` ブランチでサイトをビルドして `public/` ディレクトリに出力します。

## 開発規約

### コンテンツ整理
- **メインドキュメント**: 安定した完成済みのノートは `docs/` ディレクトリに配置します。
- **下書き**: 新規または未完成のノートは `draft/` ディレクトリに配置します。ナビゲーションバーの「下書き」リンクからアクセスできます。
- **サイドバーのソート**: サイドバーは `docusaurus.config.ts` のカスタムジェネレーターを使用して自動的にソートされます。順序は以下の通りです:
  1. `index.md`（存在する場合）
  2. カテゴリ（フォルダ）
  3. 個別ファイル（ラベル/ID のアルファベット順）

### 新しいコンテンツの追加
1. 適切なディレクトリ（`docs/` または `draft/`）に `.md` または `.mdx` ファイルを作成します。
2. 新しいカテゴリを作成する場合は、フォルダ内にラベルと位置を定義する `_category_.json` ファイルを追加します。
3. 標準の Markdown または MDX の機能を使用します。
4. Mermaid ダイアグラムは `mermaid` とラベル付けされたコードブロックを使用して埋め込めます。

### 画像
`static/img/` に画像を保存するか、Docusaurus のアセットバンドル機能を使用する場合はドキュメントに隣接するディレクトリに保存します（現在の構成では `static/` または `docs/server/gitlab_container/` のようなローカルフォルダを優先しているようです）。
